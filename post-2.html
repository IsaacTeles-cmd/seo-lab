APIs REST com FastAPI: estrutura, boas práticas e deploy

28 de Maio de 2025 6 min de leitura DevLog

FastAPI se tornou um dos frameworks Python mais populares para APIs por razões sólidas: performance próxima ao Node.js, documentação automática via OpenAPI, tipagem nativa e suporte a operações assíncronas. Neste artigo vou mostrar como estruturar um projeto de verdade — não só o "Hello World" da documentação.

Por que FastAPI?

Comparando com Flask e Django REST Framework, o FastAPI entrega alguns diferenciais importantes para projetos modernos:

  • Validação automática de dados com Pydantic v2
  • Suporte nativo a async/await com SQLAlchemy assíncrono
  • Documentação Swagger gerada automaticamente em /docs
  • Type hints que funcionam tanto para a IDE quanto para o runtime

Estrutura de projeto recomendada

Evite colocar tudo em um único main.py. Uma estrutura bem organizada facilita testes, manutenção e onboarding de outras pessoas:

projeto/
├── app/
│   ├── api/
│   │   └── v1/
│   │       ├── routes/
│   │       │   ├── auth.py
│   │       │   └── products.py
│   │       └── router.py
│   ├── core/
│   │   ├── config.py       # Settings via pydantic-settings
│   │   └── security.py     # JWT, hashing
│   ├── db/
│   │   ├── base.py         # Base declarativa
│   │   └── session.py      # Engine + get_db
│   ├── models/             # Modelos SQLAlchemy
│   ├── schemas/            # Pydantic schemas
│   ├── services/           # Lógica de negócio
│   └── main.py
├── tests/
├── alembic/
├── Dockerfile
└── docker-compose.yml

Autenticação JWT do jeito certo

Muitos tutoriais mostram JWT mas esquecem do refresh token. Em produção, usar apenas access token de longa duração é um risco de segurança. A abordagem correta separa dois tokens:

  • Access token: curta duração (15–30 min), enviado no header de cada requisição
  • Refresh token: longa duração (7–30 dias), usado apenas para gerar novos access tokens
# core/security.py
from datetime import datetime, timedelta
import bcrypt
import jwt

SECRET_KEY = "sua-chave-secreta"
ALGORITHM = "HS256"

def create_access_token(data: dict) -> str:
    payload = data.copy()
    payload["exp"] = datetime.utcnow() + timedelta(minutes=30)
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

def hash_password(password: str) -> str:
    return bcrypt.hashpw(password.encode(), bcrypt.gensalt()).decode()

def verify_password(plain: str, hashed: str) -> bool:
    return bcrypt.checkpw(plain.encode(), hashed.encode())
Nota: se você estiver no Python 3.13+, use bcrypt diretamente em vez de passlib — há incompatibilidades conhecidas com versões recentes.

Banco de dados com SQLAlchemy assíncrono

Para produção, o SQLAlchemy com operações assíncronas é o caminho. Ele permite que o servidor processe outras requisições enquanto aguarda o banco, o que aumenta significativamente o throughput:

# db/session.py
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker

DATABASE_URL = "postgresql+asyncpg://user:pass@localhost/db"

engine = create_async_engine(DATABASE_URL, echo=False)
AsyncSessionLocal = sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

async def get_db():
    async with AsyncSessionLocal() as session:
        yield session

Deploy com Docker

Um Dockerfile mínimo e funcional para FastAPI:

FROM python:3.12-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .
EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

Para deploy gratuito, o Render aceita imagens Docker direto do GitHub. É exatamente como o projeto de portfólio referenciado na home deste blog foi publicado.

Testes com pytest

Cobertura de testes é o que separa um projeto de portfólio de um projeto de produção. Use o TestClient do FastAPI junto com banco de dados em memória para testes rápidos:

# tests/test_auth.py
from fastapi.testclient import TestClient
from app.main import app

client = TestClient(app)

def test_register_user():
    response = client.post("/api/v1/auth/register", json={
        "email": "test@example.com",
        "password": "senha123"
    })
    assert response.status_code == 201
    assert "id" in response.json()

Conclusão

FastAPI com a estrutura certa, JWT com refresh token, SQLAlchemy assíncrono e Docker resulta em uma API que tem cara de projeto profissional — não de tutorial. Esse combo é exatamente o que recrutadores de back-end Python esperam ver em um portfólio sólido.
